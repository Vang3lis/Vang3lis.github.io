SMB 环境搭建

[toc]

简介

尝试搭建SMB的环境，以下会介绍，windows和linux如何分别做Server和Client，然后再介绍FortiGate的SSLVPN如何配置转发SMB

windows 做 SMB Server

首先在Server机器上的启用或关闭Windows功能的地方，打开SMB 1.0/CIFS文件共享支持的选项

再创建一个SMB共享文件夹，SMB-Share，在文件的属性-共享中添加用户（A），这样就开启了Windows上的SMB Server

windows 做 SMB Client

在Client的机器上，在文件系统-此电脑的位置，右键打开映射网络驱动器，按照示例\\server\share的方式，填写\\192.168.152.129\SMB-Share，再用之前设置的用户（A）的用户名、密码登录就可以进行连接了

如果要断开连接，直接在网络位置右键点击断开连接就可以了

linux 做 SMB Server

首先利用如下指令安装依赖，再通过是否存在SMB服务，判断是否装好

sudo apt-get install samba samba-dev smbclient samba-common

ls /etc/samba/smb.conf
OR samba -V
OR sudo service smbd status

添加一个名为smbusr的用户，并添加到sudo、smbashare的组里，并最后添加到smb中

sudo useradd smbusr
sudo usermod -a -G sudo smbusr
sudo smbpasswd -a smbusr

创建smbusr的路径以及改变其shell

su smbusr
mkdir /home/smbusr
chsh /bin/bash

设置一个共享的路径/home/smbusr/share，并将其设置为777

mkdir /home/smbusr/share
chmod 777 /home/smbusr/share

在 /etc/samba/smb.conf 文件的最后添加，smb share的名字以及其配置，如下所示，注意develop_share才是真正的smb share的名字，而不是共享的文件夹的名字

[develop_share]
    valid users = smbusr
    path = /home/smbusr/share
    browseable = yes
    writable = yes
    available = yes
    public = yes
valid users 允许访问的用户
path 是共享目录路径
browseable 是否可浏览
writeable 是否可写
available 是否可用
public 是否允许guest访问

重启 SMB Server

sudo service smbd restart

通过 sudo service smbd status和demsg 查看状态以查看无法连接的情况

linux 做 SMB Client

先安装依赖

sudo apt-get install smbclient
sudo apt-get install cifs-utils

再创建mount的文件夹mkdir SMB-Share

最后mount就可以了，-t指定类型，-o指定参数

sudo mount -t cifs //192.168.152.129/SMB-Share ./SMB-Share -o user=vangelis,pass=xxx
